캐시(cache)
미리 만들어진 데이터를 임시로 저장하는 공간
하드웨어(CPU,메모리,디스크등)부터 운영체제,응용프로그램에 이르기까지 다양한
IT분야에서 캐시 매커니즘을 이용하여
한정된 자원을 효율적으로 관리하고,성능 향상을 목적으로 사용

일반적으로 웹 앱이 제공하는 기능의 20퍼가 자원의 80퍼를 소비
자주 사용하는 데이터는 그 다음에 또 사용될 확률이 매우 높음
집중적으로 사용되는 부분에 캐시 매커니즘을 도입하면 전체적인 시스템 성능 향상과
효율적인 자원 사용을 가능하게 함

단순한 아키텍처에 4개정도의 캐싱 가능한 지점이있음

1. 브라우저 -> 웹서버
2. 웹서버 -> VIEW(서블릿,JSP)
3. VIEW -> 비즈니스로직
4. 비즈니스로직 -> 데베

1번지점에서는 HTTP가 제공하는 캐싱 메커니즘 존재( 304)

2.번지점에선 페이지 요청결과(HTML,XML,JSON,...)자체를 캐싱하는 기능

3.번지점에선 서버쪽 연산결과를 메모리나 파일등에 넣고 계속 쓰는 것
프로그래밍을 수반

4. DB질의 결과를 캐싱 Ibatis나 hibernate같은 프레임워크 지원

캐싱지점이 브라우저와 가까울 수록 성능은 뛰어남
뒤로 갈수록 활용성이 좋아짐

분산캐시는
분산 환경에서 캐시 공유를 통해 다른 JVM의 애플리케이션에서도 캐시 데이터를 공유
캐싱된 데이터를 여러 시스템에서 활용할 수 있어 퍼포먼스 향상에 도움이 되고
효율적으로 자원을 관리
